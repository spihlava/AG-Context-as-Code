# Tech Stack

## Runtime
- Bun (latest)

## Framework
- Vanilla HTML/CSS/JavaScript (no framework needed for this simple game)

## Language & Typing
- JavaScript (ES6+)

## Key Libraries
None - vanilla implementation

## Data & State
- **Database**: None
- **State Management**: In-memory game state object
- **Caching**: None

## Patterns to Follow
- Functional programming where possible
- Separation of concerns (game logic, UI, state)
- Pure functions for game logic (testable)

## Anti-Patterns to Avoid
- No jQuery or heavy DOM manipulation libraries
- No global variables (use module pattern or closures)
- No inline event handlers in HTML

## Development Tools
- **Package Manager**: None needed (vanilla)
- **Linting**: None for this simple project
- **Formatting**: Consistent 2-space indentation

## Specific Rules
- Game logic must be pure functions (no side effects)
- UI layer should be cleanly separated from game logic
- Use semantic HTML5 elements
